# 天枢IoT Flutter低代码工具 - 开发计划

## 一、项目概述

### 1.1 项目名称
**天枢IoT Studio** - Flutter代码生成式低代码开发工具

### 1.2 项目定位
这是一个**代码生成工具**，而非运行时低代码平台：
- 用户通过拖拽控件设计界面
- 工具生成**可读性高、结构清晰**的Flutter源代码
- 用户可导出代码进行**二次开发和定制**
- 着重支持**移动端**（iOS/Android），同时兼容桌面端

### 1.3 核心价值
- **快速原型**：拖拽生成基础App框架
- **代码可控**：生成的代码清晰可读，便于二次开发
- **移动优先**：针对手机端优化的控件和布局
- **平台集成**：深度集成天枢IoT平台功能

### 3.4 AI Agent 模块

#### 3.4.1 功能概述
- 独立的AI对话标签页
- 支持多种AI接入方式
- 设备控制和数据查询能力
- 对话历史管理

#### 3.4.2 AI接入方式

| 模式 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 平台在线 | 直接使用天枢IoT平台配置的AI | 无需配置、统一管理 | 依赖平台 |
| 平台代理 | 通过平台API网关代理AI请求 | 统一计费、安全 | 有延迟 |
| 本地直连 | 用户自行配置API Key本机调用 | 低延迟、灵活 | 需自行管理 |

#### 3.4.3 AI工具能力（Function Calling）

基于天枢IoT平台已有的Agent工具系统：

| 工具 | 功能 | 参数 |
|------|------|------|
| device_control | 控制设备 | device_id, command, params |
| device_query | 查询设备状态 | device_id |
| data_query | 查询历史数据 | device_id, limit |
| config_get | 获取配置 | device_id |
| config_set | 设置配置 | device_id, key, value |

## 四、增强功能

### 4.1 子账户系统（需修改天枢IoT平台）

#### 功能说明
主账户可创建子账户，子账户使用生成的App访问受限资源。

#### 平台端新增API
- `POST /auth/sub-accounts` - 创建子账户
- `GET /auth/sub-accounts` - 子账户列表
- `PUT /auth/sub-accounts/<id>` - 更新子账户
- `DELETE /auth/sub-accounts/<id>` - 删除子账户

#### 子账户权限配置
| 权限项 | 描述 |
|-------|------|
| 可访问设备 | 指定可查看/控制的设备列表 |
| 可用Agent | 指定可使用的AI Agent |
| 数据权限 | 只读/读写 |
| 有效期 | 账户过期时间 |

### 4.2 生成App的设置页面（可配置）

#### 设置页面配置项（开发工具中配置）
用户在Studio中可选择生成的App包含哪些设置项：

| 设置模块 | 可选 | 描述 |
|---------|------|------|
| 服务器连接 | 是 | 服务器地址、超时等 |
| 用户登录 | 是 | 启用子账户登录功能 |
| AI本地配置 | 是 | API Key、模型、参数等 |
| 从平台拉取AI配置 | 是 | 使用平台配置的AI |
| 主题设置 | 是 | 亮色/暗色/跟随系统 |
| 语言设置 | 是 | 多语言支持 |
| 数据刷新 | 是 | 刷新间隔配置 |
| 关于页面 | 是 | App版本信息 |

#### AI本地配置详情
生成的App中，用户可在设置页面配置：

| 配置项 | 描述 |
|-------|------|
| API提供商 | OpenAI/Claude/通义千问/DeepSeek/自定义 |
| API Key | 用户自己的密钥 |
| Base URL | 自定义API端点（兼容OpenAI格式） |
| 模型名称 | 如gpt-4、claude-3等 |
| 温度 | 0-2，控制随机性 |
| 最大Token | 回复长度限制 |
| 系统提示词 | 可选自定义 |

### 4.3 设备配对功能（可选，需修改平台）

#### 使用场景
| 场景 | 是否需要配对 |
|------|-------------|
| DIY自用 | 否，直接绑定设备ID |
| 产品级应用 | 是，用户扫码/输入配对码绑定设备 |

#### 配对流程
1. 设备生成配对码（6位数字，有效期5分钟）
2. 用户在App输入配对码或扫描二维码
3. 平台验证并绑定设备到用户账户

#### 平台端新增API
- `POST /devices/pairing-code` - 设备请求生成配对码
- `POST /devices/pair` - 用户提交配对码绑定设备
- `DELETE /devices/<id>/binding` - 解绑设备

### 4.4 设备列表功能（可选）

#### 使用场景
| 场景 | 是否需要设备列表 |
|------|-----------------|
| 单设备控制 | 否，App直接控制指定设备 |
| 多设备用户 | 是，用户可切换/管理多个设备 |

#### 功能详情
- 显示用户已绑定的所有设备
- 设备在线/离线状态显示
- 切换当前控制的设备
- 设备重命名、解绑操作

### 4.5 项目管理

- **项目创建**：新建空白项目或从模板创建
- **项目导入/导出**：JSON格式项目文件
- **云端同步**：与天枢IoT平台同步项目
- **版本历史**：本地版本快照

### 4.6 模板系统

- **设备模板同步**：从平台拉取设备模板
- **自动生成控件**：根据模板数据点自动生成对应控件
- **命令面板**：根据模板命令定义生成控制按钮

### 4.7 数据绑定系统

- **可视化绑定**：点选设备和字段完成绑定
- **表达式支持**：简单计算和格式化
- **条件样式**：根据数据值改变控件样式

### 4.8 告警系统

- **阈值告警**：数据超限提醒
- **离线告警**：设备离线通知
- **告警历史**：告警记录查看

### 4.9 快捷操作

- **快捷键**：常用操作键盘快捷键
- **右键菜单**：上下文操作菜单
- **命令面板**：Ctrl+P快速命令

### 4.10 离线模式

- **本地缓存**：设备列表和配置缓存
- **离线编辑**：无网络时编辑项目
- **同步队列**：恢复连接后同步操作

## 八、总结

天枢IoT Studio 将提供：

1. **代码生成式开发** - 生成可读性高的Flutter源码，支持二次开发
2. **移动端优先** - 针对iOS/Android优化，兼容桌面端
3. **可配置设置页面** - 生成的App设置项可自定义
4. **AI本地配置** - 用户可在App中配置自己的AI API
5. **子账户系统**（可选）- 主账户创建子账户，权限可控
6. **设备配对**（可选）- 产品级应用支持扫码配对
7. **设备列表**（可选）- 多设备用户管理和切换
